Penjelasan singkat

BANYU - Saya menamainya, yes mulai dari backend, frontend, db, mobile, logo, sampai nama pun saya urus sendirian. Awal mula diluncurkan, karena sekalian hari jadi / HUT perusahaan pada tanggal 30 Januari 2019, sehingga saya diminta untuk membuat aplikasi Android ini, dengan tujuan sebagai jembatan informasi antara pelanggan dan perusahaan. Pada tahap pertama "yang penting jadi", saya menggunakan IONIC sebagai teknologinya. Setelah 4 bulan aplikasi berjalan, saya berinisiatif sendiri merefactoring seluruh kode dari IONIC ke Flutter, proses memakan waktu sekitar 3-4 bulan untuk refactoring. Untuk tampilan dan fungsi sudah mengalami berbagai perubahan dan tambahan sesuai keingin dari bagian yang membutuhkan. Sampai saat ini, aplikasi berjalan dan sudah diinstal lebih dari 7.000 devices (berdasarkan dari laporan android console). Aplikasi Banyu, menggunakan sistem satu aplikasi untuk dua keperluan, yaitu bisa sebagai aplikasi pelanggan (publik) atau bisa sebagai aplikasi karyawan (privat). Jika login di aplikasi menggunakan NIPP karyawan, otomatis aplikasi berubah menjadi aplikasi privat.
Semua hak akses bersifat dinamis, yaitu bisa menggunakan acuan KODE jabatan karyawan atau per NIPP/Karyawan masing-masing, baik itu akses lihat dan atau akses eksekusi.

Fitur / Menu
Pelanggan : - Cek tagihan
- Pembayaran (BRI Virtual Account / BRIVA)
- Pengaduan
- OTP Verifikasi pelanggan (SMS Masking / WA)
- Perkiraan tagihan bulan ini
- Simulasi tarif
- Informasi gangguan / perbaikan
- Riwayat pelanggan
- Notifikasi jika terbit rekening
- Notifikasi jika aduan telah ditangani dan atau selesai
- Notifikasi jika pembacaan meter telah dilaksanakan (sehingga bisa cek perkiraan tagihan bulan ini)
Karyawan : - Login ke aplikasi Banyu ini menggunakan NIPP karyawan dengan sistem OTP (SMS Masking / WA), login dilakukan sekali
- Login QR, login ke sistem utama app.pdambatang.co.id hanya menggunakan scan QR dengan aplikasi Banyu dari perangat masing-masing
- Riwayat kepegawaian secara detail
- Persentase bagian penagihan
- GIS / Lokasi pelanggan by koordinat dan foto rumah pelanggan
- Cek rekap absensi, sehingga terlihat jelas potongan dan detail absensi setiap hari
- Input data lokasi GIS dan foto rumah pelanggan
- Sistem pembacaan / pencatatan meter pelanggan
- Daftar aduan dari pelanggan
- Rangkuman rekening (perkiraan pendapatan perusahaan) yang telah dibaca / dicatat oleh petugas
- Notifikasi jika aduan dari pelanggan / CS kepada masing-masing petugas yang mempunyai wewenang, baik akses lihat atau eksekusi
Teknologi yang dipakai
Front End : - Flutter
Template : - Full Custom
Ikon : - Isometric from freepik.com
State Management : - GetX
Back End : - NodeJS
- Golang
